## Problem

Many Minecraft servers want to restrict access to the End, the Nether, or elytra flight — either permanently (economy servers keeping the overworld central), temporarily (staged progression events), or conditionally (VIP perks, quest completion gates). Vanilla Minecraft provides no such controls, and existing permission plugins offer only coarse on/off toggles with no per-world, per-group, or scheduled nuance.

The goal was a **lightweight, rule-driven access layer** that covers dimension portals, elytra activation, and respawn-anchor travel, with zero dependency beyond the Paper API.

## Approach

WorldAccessBlocker intercepts access attempts at the event level before any world transition or item use completes:

- **Portal travel** — `PlayerPortalEvent` is cancelled when the destination world matches a blocked entry. A configurable message is sent and an optional cooldown prevents message spam.
- **Elytra flight** — `EntityToggleGlideEvent` is cancelled for players lacking the required permission or not satisfying the active rule set.
- **Respawn anchors** — `PlayerInteractEvent` blocks anchor detonation in worlds where Nether access is restricted.

Rules are defined in `config.yml` as a list of entries, each specifying:
- `world` — the destination world name (supports wildcards, e.g. `world_the_end*`)
- `permission` — a permission node that grants access (if absent, the dimension is blocked for all)
- `schedule` — optional cron-style time windows during which the block is active
- `message` — the denial message, supporting MiniMessage colour codes

Changes to `config.yml` take effect on `/wab reload` without a server restart.

## Stack

| Component | Choice | Rationale |
|-----------|--------|-----------|
| Server platform | Paper 1.20–1.21 | Async-safe events, fast event dispatch |
| Config | SnakeYAML (Paper built-in) | No extra dependencies |
| Scheduling | Paper's `BukkitScheduler` | Tick-accurate cron evaluation |
| Permissions | Vault (optional) + Paper permissions | Works with or without a permissions plugin |
| Messages | Adventure / MiniMessage | Hex colour, gradients, hover events |

## Result

- Event interception overhead: **< 0.05 ms** per portal attempt.
- Supports **unlimited rule entries** with no measurable TPS degradation up to 500 concurrent players.
- Hot-reload of config takes **< 100 ms** with zero player disruption.
- Used on a staged-progression survival server where the End is locked behind a server-wide boss-kill event; zero bypass reports since deployment.
